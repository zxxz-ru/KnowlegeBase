Как добавить файлы в коммит?

git 1.*
git 2.*

Как добавить чанки? Чанк - часть работы/файла. Используется если изменения не однородны в несколько фичей и необходимо разделить. Комманды: git add -i и идти по интерактивному меню или git add -p тоже самое что git add i; и выбрать 5 из меню.

Почему нельзя добавлять файлы по звезде или точке? В новой версии по точке будет добавлена только текущая dir. И будут так же добавлены файлы удалённые из гит дерева. Т.е те которые должны были остаться вне коммита. Если надо добавить всё начиная с корневой директории использовать надо :/ Есть опастность добавит файлы которые были удалены из рабочего дерева. Stackoverflow about git add

Пуш. Pushing a branch means to get the branch’s head ref from a remote repository, find out if it is an ancestor to the branch’s local head ref, and in that case, putting all objects, which are reachable from the local head ref, and which are missing from the remote repository, into the remote object database, and updating the remote head ref. If the remote head is not an ancestor to the local head, the push fails.

Чем опасен пуш с форсом? Если кто-то уже сделал изменемия в бранч они будут потеряны. git push --force переписывает историю комитов на сервере.

Что такое фастфовард? Move HEAD of merge in branch to top of branch which is merging.

Как создать и удалить ветку, локально и в origin`е. Local: git branch [branch-name], git branch -d [branch-name]
Remote: git push origin [branch-name] or git push origin [branch-name]:[new branch-name] to delete git push origin :[branch-name] пробел обязателен! или просто git push --delete [branch-name]

Как привязать локальную ветку "a1" к удаленной ветке "b2", и пушить в нее? git branch -u a1 origin/b2 [-u upstream], если не давать имя то дефолт на текущую.

Как получить список комитов из локальной ветки, которую вы только что удалили? git reflog и посмотреть историю указателей.

Сколько веток может указывать на один коммит? 'branches are free' я думаю сколько угодно веток может указывать на какой-то коммит. Пока ветка не merged она будет указывать на коммит.

Что такое мердж? Как его откатить? git reset --hard стерает всё полностью, git reset --merge оставит изменения, если уже успели поработать.

Какие бывают стратегия мерджа? resolve, recursive, octopus, ours, subtree

Что такое ребейз? Сдвиг истории всей ветки на другую ветку, история выглядит так, как будто изменения были в одной ветке. И позволяет переносить например готовые части в основную ветку. Do not rebase commits that exist outside your repository. Ещё полеьно помнить о командах позволяющих сливать вместе коммиты и позволяющих изменять что-то внутри сливающихся: git rebase -i pick, squash, edit. После того как внесли изменения в файл как обычно: git add ; git commit --amend; git rebase --continue; Если что-то пошло не так git rebase --abort;

Зачем нужен пулл по ребейзу? git pull --rebase Можно использовать вместо git fetch, git rebase, git попытается автоматически понять как смержить ветки которые были rebase на server'е.

Using git cherry-pick The command git cherry-pick commit applies the changes introduced by the named commit on the current branch. It will introduce a new, distinct commit. Strictly speaking, using git cherry-pick doesn’t alter the existing history within a repository; instead, it adds to the history. As with other Git operations that introduce changes via the process of applying a diff, you may need to resolve conflicts to fully apply the changes from the given commit . The command git cherry-pick is typically used to introduce particular commits from one branch within a repository onto a different branch. A common use is to forward- or back-port commits from a maintenance branch to a development branch.

Как перименовать коммит? 

   git commit --amend // если последний
   git rebase -i HEAD~{commit number from top}
   In todo file change pick to edit save and exit.
   git commit --amend
   git rebase --continue

Как удалить последний commit? git reset --hard HEAD~1

Как удалить коммит из середины и запушить его?

  git push [remotename] [commit SHA]:[remotebranchname]
  git rebase -onto branch-name~{commit number to start} branch-name~{commit number to end} branch-name
