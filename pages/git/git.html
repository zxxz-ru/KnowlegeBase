Как добавить файлы в коммит? git add -A (--no-ignore-removal) добавит к комиту удалённые из стажед файлы. Т.е. если есть необходимость не коммитить какие-то файлы то эту комманду лучьше не применять.

Почему нельзя добавлять файлы по звезде или точке? В новой версии по точке будет добавлена только текущая dir. И будут так же добавлены файлы удалённые из гит дерева. Т.е те которые должны были остаться вне коммита. Если надо добавить всё начиная с корневой директории использовать надо :/ Есть опастность добавит файлы которые были удалены из рабочего дерева. Пуш. Pushing a branch means to get the branch’s head ref from a remote repository, find out if it is an ancestor to the branch’s local head ref, and in that case, putting all objects, which are reachable from the local head ref, and which are missing from the remote repository, into the remote object database, and updating the remote head ref. If the remote head is not an ancestor to the local head, the push fails.

Чем опасен пуш с форсом? Если кто-то уже сделал изменемия в бранч они будут потеряны. git push --force переписывает историю комитов на сервере.

Что такое фастфовард? Move HEAD of merge in branch to top of branch which is merging.

Как создать и удалить ветку, локально и в origin`е. Local: git branch [branch-name], git branch -d [branch-name]
Remote: git push origin [branch-name] or git push origin [branch-name]:[new branch-name] to delete git push origin :[branch-name] пробел обязателен! или просто git push --delete [branch-name]

Как привязать локальную ветку "a1" к удаленной ветке "b2", и пушить в нее? git branch -u a1 origin/b2 [-u upstream], если не давать имя то дефолт на текущую.

Как получить список комитов из локальной ветки, которую вы только что удалили? git reflog и посмотреть историю указателей.

Сколько веток может указывать на один коммит? 'branches are free' я думаю сколько угодно веток может указывать на какой-то коммит. Пока ветка не merged она будет указывать на коммит.

Что такое мердж? Как его откатить? git reset --merge

Какие бывают стратегия мерджа? resolve, recursive, octopus, ours, subtree

Что такое ребейз? Сдвиг истории всей ветки на другую ветку, история выглядит так, как будто изменения были в одной ветке. И позволяет переносить например готовые части в основную ветку. После ребаза merge command всё равно нужна. Do not rebase commits that exist outside your repository.

Зачем нужен пулл по ребейзу? git pull --rebase Можно использовать вместо git fetch, git rebase, git попытается автоматически понять как смержить ветки которые были rebase на server'е.

Как пользоваться reflog? A reflog shows the local "history" of a ref - unic identifiers of Object . In other words, it can tell you what the 3rd last revision in this repository was, and what was the current state in this repository, yesterday 9:14pm.
