OOP

Object-oriented programming is an approach to designing modular reusable software systems.

Class

the definition for the data format and available procedures for a given type or class of object;

A class is a blueprint or prototype from which objects are created

**КЛАСС** - это шаблон, предназначенный для создания объектов. Класс - это именованный тип сущности из предметной области, возможно, имеющая предка (суперкласс), определенная как набор полей и методов.

Object

In Dictionary: a material thing that can be seen and touched.

In OOP: a particular instance of a Class, where the object can be a combination of variables, functions, and data structures.

Объект — это именованная сущность, одновременно содержащая данные(свойства) и поведения(методы). Слово «одновременно» в данном случае определяет ключевую разницу между ООП и другими методологиями программирования. Например, при процедурном программировании код размещается в полностью отдельных функциях или процедурах.

Interface

An interface is a contract between a class and the outside world. When a class implements an interface, it promises to provide the behavior published by that interface.

Encapsulation

Can't realy give short definition. Condition of Class to hide internal data representation and implementation details of methods inside of the Class, hiding Object's State. Leads to Information Hiding, and weak coupling of models through programming to inteface principle.

Association

Тип слабой зависимости две сущьности сушествуют отдельно друг от друга. Объект имеет доступ к другому через указатель. Ассоциация – это когда один класс включает в себя указатель на другой класс в качестве одного из полей.

Aggregation

АГРЕГАЦИЯ - форма отношений классов. Подразумевающая создание нового класса из уже существующих классов путём их включения в свой состав. Составная часть передается через параметры конструктора.

However, if the enclosing class is deleted, the aggregated object wouldn't be destroyed. Note that objects participating in an aggregation relationship cannot have cyclic aggregation relationships, i.e., a whole can contain a part, but the reverse is not true.

Composition

Композиция (агрегирование по значению) — тип строгой зависимости и более строгий вариант агрегирования, когда включаемый объект может существовать только как часть контейнера. Если контейнер будет уничтожен, то и включённый объект тоже будет уничтожен. Композиция подразумевает создание объектов с использованием других объектов (has-a relationship).

Cohesion

Связность.
Сцепление, связь, сплочённость. Думаю о связанности внутри модуля.

Coupling

Связанность
Спаривание, соединение. Думаю о соединении модулей между собой.
Coupling is a measure of the degree of dependence between two items – the more highly coupled two things are the greater the chance that a change in one will require a change in another. Coupling is the “root of all evil” when it comes to software development, and the more things that your database schema is coupled to the harder it is to maintain and to evolve.

Decomposition

Disintegration, разложение. Разложение сложного модуля или объекта обладаюшего сложным функционалом и реализацией, отвечающего за большой объём функционала приложения на меньшие типы для улучьшения инкапсуляции, модульности, полиморфизма в соответствии с принципами SOLID.
