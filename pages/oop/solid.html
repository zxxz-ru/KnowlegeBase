S (Single responsibility) - single responsiblitity principle
O (Open/closed) - opent to extension close to modification
L (Liskov substitution) - types can be interchanged (Polymorphism)
I (Interface segregation) - many specific interface are better than one huge
D (Dependency inversion) - from abstraction to realisation, realisation depends on abstraction, but not vice versa.

S Class better be simple. Decompose application better. Easy to support and modify.

O Battle of composition vs inheritance, extension here means extend/change behaviour.

L Type can be replaced with objects of type's subtype or with type which is sharing common interface, without altering any of the desirable properties of the program (correctness, task performed, etc.).

I

D one should "depend upon abstractions, [not] concretions.
