Composition is what type does.

  • Flexibility. Easier change of behaior through delegation to member types.
  • Program to Interface principle. Use of Polymorphism. Like in Strategy Pattern.
  • Ability to change behaviour at run time, no need to rebuild.
  • More complains with open/close principle of SOLID.
  • Quicker instantiation, less load time. Member types can be instantiated in other Threads or earlier or cashed.
  • More natural and somehow easier to comprehend decomposition.
  • None or less unwanted methods.
  • No namespace pollution, no shadowed methods.
  • No need for instances, can use static methods. Smaler memory load.

Inheritance is more of what type is.

  • DRY Principle.
  • Specification and fine grade subtypes of same family of types.
  • Easier architecture, more linear.
  • Less latency during execution, method calls are going to same location in memory.
  • Can use Polymorphism to group subtypes of one common type, Generic use, like in Java's Collections
