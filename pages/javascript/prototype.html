__proto__ свойство анонимного объекта (анонимный объект - это прототип объекта). __proto__ является accessor property ( has get and set in propertyDescription Object ).

Object.getPrototypeOf(object1) вернёт то на что указывает свойство __proto__ анонимного объекта. instanceObject.isProtorypeOf(anotherObject) вернёт true если __proto__ of anotherObject points to анонимный объект instanceObject.Каждый объект автоматически имеет это свойство. Цепочка прототипов объектов использует это свойство для связи между прототипами объектов. const obj = {} уже сразу наследует методы Object по прототипной цепочке. Его свойство __proto__ == Object (анонимный объект)
  • Что будет если создать объект и мне нужен прототип другог объекта?

    Object.setPrototypeOf(obj, prototype) рекомендуемый метод. Можно ещё obj.__proto__ = anotherObj, но могут возникнуть проблемы с поддержкой браузеров.

  • Как передать прототип в объект при создании?

    Object.create(proto[, propertiesObject]) т.е. примерно так : Rabbit.prototype = Object.create(Animal.prototype); теперь Rabbit унаследовал все enumerable свойства объекта Animal. Для того чтобы при создании объекта с new необходимо пересохранить свойство constructor так: Rabbit.prototype.constructor = Rabbit; 

              function Employee (name, section) {
          this.name = name;
          this.section = section;
          }

          function Manager (name , section, report) {
          Employee.call(this, name, section); // call function Employee within this context, pass arguments
          // Employee.apply(this, arguments); // the same as above, only use arguments property of function and call them as [] 
          this.report = report;
          }

          Manager.prototype = Object.create(Employee.prototype);
          Manager.prototype.constructor = Manager;
          console.log(new Manager("Will", "backoffice", "bla-bla").name); // Will

prototype свойство функции-конструктора. Указывает на тот же самый анонимный объект который создаётся в момент инициализации объекта.

  • Что же именно такое prototype и для чего он вообще нужен? Можно же использовать свойство __proto__. Во первых неполная поддержка браузерами, рекомендации в документации, необходимость создания объекта перед тем как можно будет использовать свойство, и самое главное они указывают на разные объекты! prototype указывает на анонимный объект который прототип функции/класса, а __proto__ указывает на анонимный объект от которого унаследовали.
  • Как используется prototype? Для доступа к объекту прототипа (анонимному объекту). Можно переопределять и записывать новые или изменять существующие свойства объекта прототипа по этой ссылке.
  • prototype - это статические свойства? Лично я думаю, что да. По крайней мере в классическом понимании статических методов/свойств. Все объекты чьим прототипом является прототип имеют доступ к свойствам и видят изменения в этом объекте.

Every object can have another object as its prototype. Then the former object inherits all of its prototype’s properties. An object specifies its prototype via the internal property [[Prototype]]. The chain of objects connected by the [[Prototype]] property is called the prototype chain. в Функции конструкторе в переменной prototype тот указатель на то что будет записано в прототип (__proto__) создаваемого объекта.
