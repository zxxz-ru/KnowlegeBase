Think about Heap, Events Queue, Stack. Message when the job is done in Events Demultiplexer goes to Events Queue, when/if Stack is empty function callback goes to execution. While executing messages from Events Queue this execution will have it's own Stack. See external links. setTimeout is one of functions that introduce another player - WebAPI, which is delaying execution of callback function. And is not part of Javascript, functionality is provided by Browser.
