const and let

First must undertand scoping in JS. First article and second one have good explanation about scope in JS. Historically it was defined as global and function scope. var made variable global if outside of function or global to function in inside of function. const and let are block scoped {}.

arrow function

this in arrow function is taken form outer context. Example on fiddle.net
Can not call function with new key word.
No own arguments, will get then from enclosing function. Example on fiddle.net
Kак только тело функции оборачивается в {…}, то её результат уже не возвращается автоматически, необходим return.
() => {} это не то же самое, что .bind(this) Есть тонкое различие между функцией стрелкой => и обычной функцией, у которой вызван .bind(this): Вызовом .bind(this) мы передаём текущий this, привязывая его к функции. При => привязки не происходит, так как функция стрелка вообще не имеет контекста this. Поиск this в ней осуществляется так же, как и поиск обычной переменной, то есть, выше в замыкании. До появления стандарта ES-2015 такое было невозможно.

String templates

String templates are made with backticks and are one of basic types of javascript. In string templates string formatting is preserved. No need to type \n for new line. Here is good discussion on Stackoverflow about difference between single quoted and double quoted strings.
